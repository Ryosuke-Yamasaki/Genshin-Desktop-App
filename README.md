# GenshinDesktopApp

## 目的

原神の所持聖遺物の中から 1 ローテーションの総ダメージが最大となるような聖遺物を選定する。

## 要件定義

### 機能要件

| 機能名                     | 内容                                                                                                      |
| -------------------------- | --------------------------------------------------------------------------------------------------------- |
| 聖遺物管理システム         | 所持聖遺物のデータを登録・編集・削除する                                                                  |
| ゲーム内データ管理システム | キャラクター・武器・聖遺物のデータを登録・編集・削除する                                                  |
| パーティ編成管理システム   | 任意の 4 キャラクターのグループをパーティとし、そのパーティの登録・編集・削除する                         |
| ローテーション管理システム | あるパーティ編成のスキルローテーションを登録・編集・削除する                                              |
| ダメージ計算システム       | 1 ローテーションでの総ダメージ量を計算する                                                                |
| 聖遺物最適化システム       | パーティ編成で 1 キャラクターを指定し、1 ローテーションでの総ダメージ量が最大となるような聖遺物を選定する |
| 結果保存システム           | ダメージ計算した結果を登録・編集・削除する                                                                |
| 比較システム               | パーティ編成での比較やローテーションでの比較をする                                                        |

### 非機能要件

- Java
- デスクトップアプリケーション
- Swing
- Maven
- 保存データは json 形式
